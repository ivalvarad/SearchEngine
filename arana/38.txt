</tr>
<tr>
<td style="font-size:105%;background: #EAF4FF;">
<ul>
<li><b>This page is <u>only</u> for questions about <span style="color:red;"><i>how to use or edit</i></span> Wikipedia.</b> For other types of questions, use the <a href="/wiki/Special:Search" title="Special:Search">search box</a> or the <a href="/wiki/Wikipedia:Reference_desk" title="Wikipedia:Reference desk">Reference desk</a>.</li>
<li><b>Do <span style="color:red;">not</span> provide your email address</b> or any other contact information. Answers will be provided <b>on this page only</b>.</li>
<li>We are all volunteers, so sometimes replies can take some time. Please be patient. Check back on this page to see if your question has been answered.</li>
</ul>
<hr />
<ul>
<li><b>New users:</b> While this is a good place to ask questions, new users may prefer to ask for help at <b><a href="/wiki/Wikipedia:Teahouse" title="Wikipedia:Teahouse">the Teahouse</a></b>, an area specifically for new users to get help with editing, article creation, and general Wikipedia use, in a friendly environment.</li>
</ul>

<dd>Yes, double curly brackets <code>{{...}}</code> is either <a href="/wiki/Wikipedia:Transclusion" title="Wikipedia:Transclusion">transclusion</a> or <a href="/wiki/Help:Magic_words" title="Help:Magic words">magic words and parser fucntions</a>, while triple curly brackets <code>{{{...}}}</code> is template parameters. More than three is nesting of double or triple cases (or an error). It's very common for ending brackets and can make template code difficult to read when there are many. A single curly bracket <code>{...}</code> has no meaning in syntax and is just displayed as text. Three straight brackets <code>[[[...]]]</code> has no meaning. Double straight brackets <code>[[...]]</code> are not usually nested except in file descriptions where four straight ending brackets can occur with a wikilink at the end of the file decription <code>[[File:...|Description [[...]]]]</code>. <a href="/wiki/User:PrimeHunter" title="User:PrimeHunter">PrimeHunter</a> (<a href="/wiki/User_talk:PrimeHunter" title="User talk:PrimeHunter">talk</a>) 12:08, 18 October 2015 (UTC)</dd>

<h2><span class="mw-headline" id="Nick_Bosustow">Nick Bosustow</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Wikipedia:Help_desk&amp;action=edit&amp;section=46" title="Edit section: Nick Bosustow">edit</a><span class="mw-editsection-bracket">]</span></span></h2>
<p>Nick Bosustow was an Oscar wining animation produce, and son of Stephen Bosustow.Born in Los Angeles March 28 1940 Married to Julie Jonas April 1971 and had 2 daughters Nicole 1972 and Jennifer 1977.
